Cette page documente **uniquement** la forme de la spécification JSON
(`DataFlowSpec`) passée à la prop `spec`, générée à partir du JSON Schema. Pour
les **composants et fonctions exportés** (props de `<DataFlowPlayer>`, icônes
personnalisées, coloration syntaxique), voir
[Composants et API JavaScript](./components.mdx).

:::info JSON Schema disponible

Le JSON Schema brut de `DataFlowSpec` est exposé via
`react-dataflow-animator/schema.json` (draft-07). Il peut être utilisé
pour la validation Ajv côté utilisateur, l'auto-complétion d'éditeur, ou
la génération de formulaires. Voir la section
[Auto-complétion JSON dans VS Code](../installation#autocomplete-json-vscode)
pour la configuration VS Code.

:::

:::note Divergence TS ↔ schéma JSON pour `language`

Le type TypeScript de `language` est `HighlightLanguage | (string & {})` : n'importe
quelle chaîne est acceptée à la compilation pour ne pas casser les consommateurs qui
utilisent un highlighter personnalisé. Cependant, le schéma JSON publié est **plus
strict** : le script `schema-patches.mjs` retire la branche `string` libre et ne
conserve que l'énumération `HighlightLanguage`. Un langage hors de cette énumération
passe le compilateur TypeScript mais est **rejeté par Ajv** lors de la validation du
schéma.

:::
