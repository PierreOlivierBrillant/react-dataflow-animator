title: Types d'actions

Les types d'actions suivants sont disponibles. Tous partagent un jeu de champs
communs ([cycle de vie](../concepts/timeline.mdx) : `id`, `duration`,
`wait_for`, `keep_until`, `keep_until_next`, `keep_until_end`).

## `move`

Déplace un objet dynamique de `from` vers `to`.

```ts
{
  type: 'move',
  object: 'req',     // id d'un packet
  from: 'browser',   // id d'un node
  to: 'api',         // id d'un node
  duration: 600,
}
```

Produit deux points d'arrêt : apparition et arrivée. Par défaut, l'objet
disparaît à la fin de son animation.

## `arrow`

Trace une flèche animée entre deux nœuds (dessin progressif).

```ts
{
  type: 'arrow',
  from: 'a',
  to: 'b',
  style: 'dashed',         // 'solid' | 'dotted' | 'dashed' | 'animated'
  arrow_head: 'both',      // 'forward' | 'backward' | 'both' | 'none'
  text: 'requête',
}
```

Pour des flèches **permanentes** (décor affiché dès l'initialisation),
utilisez plutôt le tableau racine `connections`.

## `parallel`

Encapsule des actions enfants exécutées au même instant.

```ts
{
  type: 'parallel',
  actions: [
    { type: 'move', object: 'p1', from: 'a', to: 'b' },
    { type: 'arrow', from: 'c', to: 'd' },
  ],
}
```

Les `wait_for` et `keep_until` à l'intérieur d'un `parallel` continuent de
fonctionner normalement.

## `loading`

Affiche un spinner de chargement sur un nœud cible (simule un traitement).

```ts
{ type: 'loading', object: 'db', duration: 900 }
```

## `set_content`

Mute le contenu d'un nœud. Trois modes :

- **`code`** : terminal sombre avec coloration Prism, sans barre d'URL ; la
  police s'ajuste pour faire tenir la ligne la plus longue sans wrap.
- **`text`** : fenêtre de navigateur avec barre d'URL paramétrable.
- **`image`** : image affichée dans une fenêtre.

```ts
{
  type: 'set_content',
  object: 'editor',
  content: {
    type: 'code',
    language: 'javascript',
    content: 'const add = (a, b) => a + b;',
  },
}
```

## `comment`

Affiche une bulle de texte en fondu près d'un nœud.

```ts
{
  type: 'comment',
  object: 'browser',
  text: 'L\'utilisateur clique',
  duration: 400,
}
```

## `highlight`

Surligne (halo pulsé) un nœud statique **ou** une connexion permanente,
identifiés par leur `id`.

```ts
{ type: 'highlight', object: 'db' }
```

## `wait`

Insère un temps mort : rien ne se passe pendant `duration` ms (défaut :
1000). L'action ne produit aucun élément à l'écran ; elle se contente de
décaler l'étape suivante. Le contenu déjà posé (via `keep_until_next`)
reste affiché **pendant** l'attente, ce qui fige l'image le temps voulu.

```ts
{ type: 'wait', duration: 1500 }
```

Utile pour laisser le spectateur lire une étape avant de poursuivre, ou
pour rythmer une lecture en `autoPlay`.
